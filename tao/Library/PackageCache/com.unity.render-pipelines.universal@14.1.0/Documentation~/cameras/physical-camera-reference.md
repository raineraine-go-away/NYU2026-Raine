# Physical Camera 参考

物理相机属性使得 URP 相机能够模拟现实世界的相机。这些属性对应于现实世界相机的功能，并以相同的方式工作。

有关如何使用这些属性来创建您想要的相机效果的更多信息，请参阅 [使用物理相机](xref:PhysicalCameras)。

> [!NOTE]
> 当使用物理相机时，Unity 会根据以下属性计算视场（Field of View）：
>
> * **Sensor Size**
> * **Focal Length**
> * **Shift**

物理相机属性分为以下几部分：

* [相机机身](#camera-body)
* [镜头](#lens)
* [光圈形状](#aperture-shape)

## 相机机身

| **属性** | **描述** |
| -------- | -------- |
| **Sensor Type** | 指定您希望相机模拟的现实世界相机格式。当选择相机格式时，Unity 会自动将 **Sensor Size** > **X** 和 **Y** 属性设置为正确的值。<br/><br/>URP 提供了以下相机格式预设：<ul><li>**8mm**：<ul><li>**X**: 4.8</li><li>**Y**: 3.5</li></ul></li><li>**Super 8mm**：<ul><li>**X**: 5.79</li><li>**Y**: 4.01</li></ul></li><li>**16mm**：<ul><li>**X**: 10.26</li><li>**Y**: 7.49</li></ul></li><li>**Super 16mm**：<ul><li>**X**: 12.522</li><li>**Y**: 7.417</li></ul></li><li>**35mm 2-perf**：<ul><li>**X**: 21.95</li><li>**Y**: 9.35</li></ul></li><li>**35mm Academy**：<ul><li>**X**: 21.946</li><li>**Y**: 16.002</li></ul></li><li>**Super-35**：<ul><li>**X**: 24.89</li><li>**Y**: 18.66</li></ul></li><li>**35mm TV Projection**：<ul><li>**X**: 20.726</li><li>**Y**: 15.545</li></ul></li><li>**35mm Full Aperture**：<ul><li>**X**: 24.892</li><li>**Y**: 18.669</li></ul></li><li>**35mm 1.85 Projection**：<ul><li>**X**: 20.955</li><li>**Y**: 11.328</li></ul></li><li>**35mm Anamorphic**：<ul><li>**X**: 21.946</li><li>**Y**: 18.593</li></ul></li><li>**65mm ALEXA**：<ul><li>**X**: 54.12</li><li>**Y**: 25.59</li></ul></li><li>**70mm**：<ul><li>**X**: 52.476</li><li>**Y**: 23.012</li></ul></li><li>**70mm IMAX**：<ul><li>**X**: 70.41</li><li>**Y**: 52.63</li></ul></li><li>**Custom**：<ul><li>手动设置 **X** 和 **Y** 值</li></ul></li><ul/><br/>如果您手动更改 **Sensor Size** 值，Unity 会自动将此属性设置为 **Custom**。 |
| **Sensor Size** | 设置相机传感器的大小（单位：毫米）。<br/><br/>当选择 **Sensor Type** 时，Unity 会自动设置 **X** 和 **Y** 值。 |
| &#160;&#160;&#160;&#160;**X** | 相机传感器的横向尺寸。 |
| &#160;&#160;&#160;&#160;**Y** | 相机传感器的纵向尺寸。 |
| **ISO** | 相机传感器的光敏度。 |
| **Shutter Speed** | 相机传感器曝光的时间。 |
| &#160;&#160;&#160;&#160;**Unit** | **Shutter Speed** 的计量单位。<br/><br/>可选项：<ul><li>**Second**</li><li>**1/Second**</li></ul> |
| **Gate Fit** | 改变 **resolution gate**（游戏视图的大小/宽高比）相对于 **film gate**（物理相机传感器的大小/宽高比）的选项。<br/><br/>有关分辨率门和胶片门的更多信息，请参阅关于 [Physical Cameras](https://docs.unity.cn/cn/tuanjiemanual/Manual/PhysicalCameras.html) 的文档。 |
| &#160;&#160;&#160;&#160;**Vertical** | 将分辨率门适配到胶片门的高度。<br/><br/>如果传感器宽高比大于游戏视图宽高比，Unity 会裁剪渲染图像的两侧。<br/><br/>如果传感器宽高比小于游戏视图宽高比，Unity 会在图像两侧进行超扫描。<br/><br/>选择此设置时，对传感器宽度（**Sensor Size** > **X**）的任何更改不会影响渲染图像。 |
| &#160;&#160;&#160;&#160;**Horizontal** | 将分辨率门适配到胶片门的宽度。<br/><br/>如果传感器宽高比大于游戏视图宽高比，Unity 会在图像的上下进行超扫描。<br/><br/>如果传感器宽高比小于游戏视图宽高比，Unity 会裁剪图像的上下。<br/><br/>选择此设置时，对传感器高度（**Sensor Size** > **Y**）的任何更改不会影响渲染图像。 |
| &#160;&#160;&#160;&#160;**Fill** | 将分辨率门适配到胶片门的宽度或高度（较小者）。这会裁剪渲染的图像。 |
| &#160;&#160;&#160;&#160;**Overscan** | 将分辨率门适配到胶片门的宽度或高度（较大者）。这会超扫描渲染的图像。 |
| &#160;&#160;&#160;&#160;**None** | 忽略分辨率门，仅使用胶片门。这会将渲染图像拉伸以适应游戏视图的宽高比。 |

## 镜头

| **属性** | **描述** |
| -------- | -------- |
| **Focal Length** | 相机传感器与相机镜头之间的距离（单位：毫米）。<br/><br/>较小的值会产生更宽的 **Field of View**，反之亦然。<br/><br/>当您更改此值时，Unity 会自动相应地更新 **Field of View** 属性。 |
| **Shift** | 将镜头从中心水平或垂直偏移。值为传感器大小的倍数；例如，沿 X 轴偏移 0.5 会使传感器偏移其横向大小的一半。<br/><br/>您可以使用镜头偏移来修正当相机与拍摄对象存在角度时产生的失真（例如，平行线汇聚）。<br/><br/>沿任意轴偏移镜头可以使相机视锥体变得 [倾斜](https://docs.unity.cn/cn/tuanjiemanual/Manual/ObliqueFrustum.html)。 |
| &#160;&#160;&#160;&#160;**X** | 镜头相对于相机传感器的水平偏移量。 |
| &#160;&#160;&#160;&#160;**Y** | 镜头相对于相机传感器的垂直偏移量。 |
| **Aperture** | 镜头的光圈（f-stop 或 f-number）。较小的值会提供更宽的光圈。 |
| **Focus Distance** | 启用景深效果时，物体清晰的距离。 |

> [!NOTE]
> 当物理相机属性与 [景深](../post-processing-depth-of-field.md) 后处理效果同时使用时，镜头属性会直接影响景深效果。这要求您调整景深属性和镜头属性，以创造您想要的效果。

## 光圈形状

| **属性** | **描述** |
| -------- | -------- |
| **Blade Count** | 镜头光圈叶片的数量。较高的值会使光圈形状更加圆润。 |
| **Curvature** | 镜头光圈叶片的曲率。 |
| **Barrel Clipping** | 镜头的自遮挡。较高的值会产生猫眼效应。 |
| **Anamorphism** | 相机传感器的垂直拉伸量，使传感器变得更高或更矮。较高的值会增加传感器的拉伸，以模拟变形镜头效果。 |
